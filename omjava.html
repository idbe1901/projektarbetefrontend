Vad är Javascript och Javascript events?

En del som skrivs här kan vara lite svårt att relatera till om man inte har någon erfarenhet av programmering alls sedan tidigare men jag hoppas kunna bringa lite klarhet i vad JavaScript är för något. JavaScript är ett språk som används inom programmering på webben. Det har flera likheter med andra programerings språk som till exempel C++ och Java. Men JavaScript har blivit ett av de populäraste språken att använda mycket på grund av bland annat ReactJS och Angular. Dock är en stor skillnad att JavaScript körs direkt i webbläsaren vilket ger oss tillgång till alla komponenter som bygger upp webbsidan. Programeringsspråket C++ till exempel behöver en compiler för att kunna köras vilket vi inte behöver med JavaScript utan vi har så gott som full kontroll över allt som har med webbsidan att göra direkt i webbläsaren.

Så vad använder vi egentligen JavaScript till? När man bygger en webbsida samt håller på med webbutveckling så finns det flera olika komponenter som man använder sig av. HTML används för själva dokumentstrukturen, CSS används för utseendet och sedan har vi JavaScript som används för interaktion. Så om vi till exempel har en knapp på vår webbsida så använder vi JavaScript för att se till att det faktiskt händer något när användaren trycker på knappen. Jag hoppas att detta har gett en liten insikt i vad JavaScript är och vad det används till.

Så vad är då JavaScript events? Ett event är någonting som görs alltså en händelse, nästan som ett verb, på en webbsida. Så ett event/en händelse kan vara många olika saker, allt ifrån att användaren trycker på en knapp, scrollar på sidan eller skriver på tangentbordet med mera. För att svara på ett event/en händelse så används en "event hanterare". Det är en funktion som åberopas när ett event/en händelse uppstår. Man kan ha flera hanterare för samma event/händelse och då åberopas alla när eventet/händelsen uppstår. Inom JavaScript finns det tre olika sätt att registrera en event hanterare, "inline event handlers", "DOM on-event handlers" och genom att använda "addEventListener()". Det första sättet, "inline", används sällan längre men var det enda sättet i början när JavaScript kom. DOM on-event hanterare är vanligast om man bara ska ha som mest en hanterare då det inte går att registrera mer än en på detta vis. Så det moderna sättet och det man använder sig mest av idag är alltså "addEventListener()". Så enkelt förklarat är JavaScript events händelser i webbläsaren orsakade av användaren som man sedan med hjälp av JavaScript kopplar ihop med instruktioner om vad webbläsaren ska göra med elementet på sidan.
